This Code of Conduct defines the expected behavior for all project spaces,
including issues, pull requests, discussions, reviews, docs, and chat channels.

## Our Standards

All contributors are expected to:

- communicate with empathy and respect,
- focus feedback on ideas and implementation details,
- assume positive intent and ask clarifying questions early,
- help newcomers get productive without gatekeeping,
- keep discussions constructive, factual, and actionable.

Unacceptable behavior includes:

- harassment, personal attacks, or discriminatory language,
- publishing private or identifying information without permission,
- repeated disruptive behavior after moderation guidance,
- deliberately hostile, threatening, or intimidating communication.

## Scope

This policy applies to:

- repository interactions (issues, pull requests, reviews),
- project communication channels,
- project-linked community spaces.

## Reporting and Enforcement

To report a Code of Conduct concern, contact:

- `info@livon.tech`
- include `[COC]` in the subject line

Maintainers will:

- acknowledge reports within 5 business days,
- review context and discuss actions internally,
- apply proportional actions (warning, moderation, temporary or permanent ban),
- protect reporter privacy as much as possible.

Security vulnerabilities should not be reported through this channel.
Use the private vulnerability process in [Security and Vulnerability Reporting](./security).
